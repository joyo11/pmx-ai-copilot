# DR-001: M0.1 web shell scope + install list

**Date:** 2026-07-09
**Status:** Decided

## Question
For M0.1, what does the Next.js web shell contain, and which packages do we install now vs deferring to when a feature actually needs them?

## Round-table

- **Vic (CEO):** "The shell has to make a stranger say 'this looks real' in under 3 seconds. Nav rail + dark mode + one branded landing. Skip anything that isn't visible to the demo eye."
- **Rachel (Sr EM):** "M0 is 2 days. Don't install what you don't use. Every dep is a future upgrade problem."
- **Kenji (Staff):** "shadcn is not a dep, it's a codegen — install just the primitives we know we need everywhere: Button, Card, Dialog, Sonner, Tooltip. Add more on demand. Same for Recharts; it's not needed until M1's dashboard. Framer Motion — not yet."
- **D Guy:** "The shell has to communicate 'construction PM tool' visually, not generic SaaS. Brand it in the landing hero so evaluators know what they're looking at before we ship real features."
- **Priya (Jr):** "Please make the layout obvious to me. If I clone this, I want to see where the sidebar lives, where a page goes, where a card component lives. No cleverness."
- **Milo (UI):** "Dark mode primary but respect system pref. Zinc/neutral palette. Inter for UI, tabular numerals. Sidebar rail with icon + label. Content area maxes at 1400px."

## Decision

**Install now (M0.1):**
- shadcn/ui primitives: `button`, `card`, `dialog`, `sonner`, `tooltip`, `separator`, `dropdown-menu`, `avatar`, `badge`, `input`, `label`, `sheet`, `skeleton`
- `next-themes` for dark mode
- `lucide-react` for icons
- `clsx` + `tailwind-merge` (via shadcn's `cn` helper)

**Defer (add when the feature lands):**
- Recharts → M1 dashboard
- Framer Motion → M3 polish (or when a specific animation needs it)
- Clerk SDK → M0.4 (its own task)
- react-hook-form / zod → when we hit the first form

**Shell contents:**
- App Router `layout.tsx` with `ThemeProvider` (dark default, system pref respected)
- Top-level layout: fixed left nav rail (60px collapsed, 240px expanded) + content area
- Nav rail placeholder links: Dashboard, Projects, Chat, Reports, Notifications, Settings
- Empty landing (`/`) with hero: "PMX AI — Project Risk Copilot" + one CTA "Get started" (goes nowhere yet)
- `/dashboard` route with placeholder "Coming soon" empty state, so the nav feels alive
- Global font: Geist (Next.js default is fine); use `font-variant-numeric: tabular-nums` for stats
- Metadata: title, description, OG image placeholder

**Dissent:** none material. Rachel wanted us to skip the `/dashboard` placeholder; overridden because Milo's "empty states teach" principle applies from day one.

**Escalation to Shafay:** none required — inside the "obvious" bar.
